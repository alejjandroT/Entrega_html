¿Qué son las media queries?

Los llamados media queries son una serie de funciones que incorpora CSS3 para que las páginas web se adapten "racionalmente" o adopten una maquetación diferente según la resolución en la que se muestre la página web.

Para intentar agrupar resoluciones en grupos estándares, podemos utilizar los siguientes:

  • Menores de 480px: para móviles.
  • De 481px a 768px: para tablets.
  • Mayores de 768px: para ordenadores.

¿Dónde colocarlos y cómo funcionan?

Este código CSS se suele añadir al final de los estilos. Un mediaquerie agrupa diferentes selectores e identifica en qué condiciones se deben ejecutar.

@media only screen and (max-width:320px){
  body{ background-color:red; }
  #bloque1{ font-size:12px; }
}

En este ejemplo, si la resolución es menor que 320px, el fondo será rojo. Si no se cumple, se ejecutan los selectores normales.

Sintaxis y Rango

El parámetro principal tiene dos posibilidades:

  • max-width: "Cuando la resolución sea menor que...".
  • min-width: "Cuando la resolución sea mayor que...".

También se pueden combinar rangos:

@media only screen and (min-width:320px) and (max-width:480px){ ... }

Exclusivo para móviles y tablets: Viewport

Para que el contenido no se escale automáticamente y se vea muy pequeño, debemos configurar el viewport en el <head>:

<meta name="viewport" content="width=device-width, initial-scale=1">
  • width=device-width: Adapta el contenido al ancho real del dispositivo.
  • initial-scale=1: Configura la escala inicial al 100%.
  • maximum-scale: Grado máximo de zoom permitido al usuario.

Uso del Viewport y Mediaqueries

Para que los mediaqueries funcionen correctamente es imprescindible la utilización del metadata "viewport". Sin él, el móvil escalará la página entera y los mediaqueries carecerán de utilidad.

Configuración Resultado en Móvil
Sin Mediaqueries / Sin Viewport Se escala todo para verse entero (muy pequeño).
Con Mediaqueries / Con Viewport Resultado Óptimo: El diseño cambia y el tamaño es legible.
